# 📰 보도자료 배포 일정 캘린더

사내 직원들이 보도자료 배포 일정을 한눈에 확인하고, 관리자가 일정을 등록·삭제할 수 있는 웹 캘린더입니다.

- **프론트엔드**: 정적 웹앱 (HTML/CSS/JS) — GitHub Pages 배포
- **데이터 저장**: 구글 스프레드시트 + Google Apps Script(웹앱 API)
- **권한**: 직원 = 조회 전용 / 관리자(비밀번호) = 등록·수정·삭제

## 주요 기능

- 월간 캘린더에서 배포 일정을 한눈에 확인
- 상태 4단계 색상 구분: **접수 / 수정중 / 배포예정 / 배포완료**
- 상태별 필터, 다가오는 배포 일정 요약(D-day)
- 관리자 모드(비밀번호)에서 일정 등록·수정·삭제
- 모바일 반응형

---

## 설치 및 배포 방법

### 1단계. 구글 스프레드시트 + Apps Script 백엔드 만들기

1. [구글 스프레드시트](https://sheets.new)를 새로 하나 만듭니다. (이름 예: `보도자료 일정 DB`)
2. 상단 메뉴 **확장 프로그램 → Apps Script** 를 엽니다.
3. 기본 `Code.gs` 내용을 지우고, 이 저장소의 **`apps-script/Code.gs`** 파일 내용을 전부 붙여넣고 저장(💾)합니다.
4. 함수 선택 드롭다운에서 **`setup`** 을 고르고 **실행(▶)** 합니다.
   - 처음 실행 시 권한 승인 창이 뜹니다 → **권한 검토 → 본인 계정 선택 → 고급 → (안전하지 않음)으로 이동 → 허용**
   - 실행되면 스프레드시트에 `일정` 시트가 자동 생성됩니다.
5. **관리자 비밀번호 설정** — `setPassword` 함수의 `newPassword` 값에 원하는 비밀번호를
   적고, 함수 목록에서 `setPassword` 를 선택해 **실행(▶)** 합니다. (아래 "관리자 비밀번호" 참고)
6. 우측 상단 **배포 → 새 배포** 클릭
   - 유형 선택(톱니바퀴) → **웹 앱**
   - **실행 계정: 나**
   - **액세스 권한: 모든 사용자**
   - **배포** 클릭 → 나오는 **웹 앱 URL** 을 복사합니다.
     (형식: `https://script.google.com/macros/s/AKfycb.../exec`)

> 이후 코드를 수정하면 **배포 → 배포 관리 → 편집(연필) → 버전: 새 버전 → 배포** 로 갱신하세요.

### 2단계. 프론트엔드에 API 주소 연결

`js/config.js` 파일을 열어 `API_URL` 에 위에서 복사한 웹 앱 URL을 붙여넣습니다.

```js
window.CONFIG = {
  API_URL: "https://script.google.com/macros/s/AKfycb.../exec",
};
```

### 3단계. GitHub Pages 배포

1. 이 폴더를 GitHub 저장소에 올립니다.
   ```bash
   git init
   git add .
   git commit -m "보도자료 배포 일정 캘린더 초기 버전"
   git branch -M main
   git remote add origin https://github.com/<계정>/<저장소>.git
   git push -u origin main
   ```
2. GitHub 저장소 → **Settings → Pages**
   - Source: **Deploy from a branch**
   - Branch: **main / (root)** → **Save**
3. 잠시 후 나오는 주소(`https://<계정>.github.io/<저장소>/`)를 직원들에게 공유하면 됩니다.

## 관리자 비밀번호

이 저장소는 **공개(public)** 이므로, 보안을 위해 비밀번호는 저장소 파일이 아니라
**Apps Script 편집기 안에서만** 설정합니다. (Apps Script 프로젝트는 GitHub에 올라가지 않는 별도 공간입니다.)

**설정 방법** (둘 중 하나):
- **방법 A (권장)**: Apps Script 편집기 → `Code.gs` 의 `setPassword` 함수에서
  `var newPassword = "";` 의 따옴표 안에 원하는 비밀번호를 적고,
  함수 목록에서 `setPassword` 선택 후 **실행(▶)**.
- **방법 B**: Apps Script 편집기 → **프로젝트 설정(⚙️) → 스크립트 속성** →
  속성 추가 `ADMIN_PASSWORD` = 원하는 비밀번호.

> 🔒 비밀번호는 **서버(Apps Script)에서만 검증**되며, 웹페이지(index.html / js / css)와
> 이 공개 저장소의 어떤 파일에도 저장되지 않습니다. 관리자가 입력한 값만 서버로 전송되어 대조됩니다.
> 비밀번호가 설정되기 전에는 모든 등록/수정/삭제 요청이 거부됩니다.

---

## 사용법

- **직원(조회)**: 주소 접속만 하면 캘린더가 보입니다. 일정 클릭 시 상세 확인.
- **관리자(등록/삭제)**:
  1. 우측 상단 **🔑 관리자 모드** 클릭 → 비밀번호 입력
  2. **＋ 일정 등록** 버튼 또는 달력의 **날짜 칸 클릭**으로 등록
  3. 일정 클릭 → **수정 / 삭제**

## 스프레드시트 데이터 구조 (`일정` 시트)

| id | title | date | status | department | manager | memo | createdAt | updatedAt |
|----|-------|------|--------|-----------|---------|------|-----------|-----------|
| 자동 | 제목 | YYYY-MM-DD | 접수/수정중/배포예정/배포완료 | 담당부서 | 담당자 | 비고 | 생성시각 | 수정시각 |

> 스프레드시트에서 직접 값을 수정해도 되지만, `status` 값은 반드시 네 단계 중 하나로 입력하세요.

## 데모 모드

`js/config.js` 의 `API_URL` 을 설정하기 전에는 **데모 모드**로 동작합니다.
- 데이터가 현재 브라우저(localStorage)에만 저장되어 **직원 간 공유되지 않습니다.** (화면 테스트용)
- 데모 모드에서는 서버가 없어 **아무 비밀번호나 입력하면** 관리자 모드로 전환됩니다.
  (실제 비밀번호 검증은 Apps Script 연동 후 서버에서만 이뤄집니다.)
